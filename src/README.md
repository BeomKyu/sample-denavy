# /src — 구현체 계층 (Implementation Layer)

이 디렉토리는 Denavy 3계층 아키텍처의 **컴파일 타겟 계층**입니다.

## 역할
- `/dna`와 `/spec`을 읽고 **타겟 기술 스택에 맞춰 자동 생성된 구동 가능한 소스 코드**를 담습니다.
- 명세의 '컴파일 타겟'으로, **인간은 이 코드를 직접 수정하지 않습니다.**
- 버그 발생 시 `/src`를 직접 패치하지 않고, 원인이 되는 `/spec` 또는 `/dna`를 먼저 수정합니다.

## 파일 규칙
- 확장자: 타겟 기술 스택에 따름 (예: `.py`, `.ts`, `.go` 등)
- 경로는 `/dna` 및 `/spec` 계층과 **1:1 대칭**을 이루어야 합니다.

```
예시:
  /dna/auth/login.dna   ↔   /spec/auth/login.spec   ↔   /src/auth/login.py
```

## 역방향 수정 금지 원칙 (Anti-Vibe Debugging)

버그나 에러가 발생했을 때, `/src` 코드만을 임의로 수정하는 것을 **엄격히 금지**합니다.

```
버그 발생 → /spec 논리 결함 진단 → /dna 제약 누락 진단
         → 명세 패치 후 인간 승인 → /src 재생성(Re-compile)
```
